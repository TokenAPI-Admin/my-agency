# 出单型买手笔记框架

## 总目标

买手笔记不是硬广，也不是普通分享。它要替用户做购买判断：

这个产品适合谁，为什么值得看，怎么用，哪些人不适合，买前要注意什么。

## 固定结构：1 + 3 + 1

### 1 张点击封面图

目的：让用户点进来。

封面图必须完成：

- 产品明确出现
- 一个强场景、强痛点、强对比或强审美判断
- 一句金句或封面大字
- 不要把所有卖点堆在封面

封面金句常用结构：

- “不是 X，但胜在 Y”
- “适合 X 的人，不适合 Y 的人”
- “看起来普通，真正戳人的是 X”
- “如果你在找 X，这个点先看清楚”

### 3 张试用场景图

目的：从点击走向信任。

三张图分工固定：

1. 真实使用代入：让用户看到它在真实生活里怎么出现。
2. 细节证据：展示材质、结构、设计、功能点、做工、局部细节。
3. 购买判断：讲适合/不适合、搭配、对比、尺寸、选择建议或注意事项。

每张图都要有：

- 画面主体
- 使用场景
- 构图方式
- 图上短句
- 转化目的

### 1 套正文内容

正文结构：

1. 开头钩子：先说用户为什么要看。
2. 买手判断：一句话给结论。
3. 适合谁：明确目标人群。
4. 不适合谁：降低踩雷感，增强信任。
5. 细节证据：用图片可见细节和用户给的参数支撑。
6. 使用场景：让用户想象自己拥有它。
7. 购买建议：怎么买、怎么选、买前确认什么。
8. 评论引导：引导用户问尺码、搭配、参数、同类对比。
9. 标签：品类词 + 场景词 + 人群词 + 风格词。

## 出单路径

点击 -> 代入 -> 相信 -> 判断 -> 行动

- 点击：封面和标题完成。
- 代入：第一张场景图和开头钩子完成。
- 相信：细节证据和边界表达完成。
- 判断：适合/不适合、购买建议完成。
- 行动：评论、私信、进店、下单引导完成。

## 文风要求

- 像买手，不像广告。
- 克制、自然、有判断。
- 少用形容词，多用场景和证据。
- 不写空泛口号。
- 不假装亲测。
- 不过度情绪化。
